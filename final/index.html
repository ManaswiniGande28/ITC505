Process and Creative Choices

The goal of this project was to create a working, playable version of the Lights Out game using JavaScript. The main components I needed to focus on were:

  • Game Initialization: The board needed to be initialized randomly but in a solvable state. The player should be able to toggle lights (squares), and each click would affect that square and its neighbors (up, down, left, right). The player needs to track the number of moves made and how long they've been playing.
  • Tracking Progress: I added two features for the player to track their progress: Moves Counter and Timer. These features help keep the game engaging by encouraging players to complete the puzzle with fewer moves and in less time.
  • Winning Condition: I implemented a check for when all squares are toggled off (black). Once this happens, the game checks if the player has beaten their target move count and time limit. If both conditions are satisfied, the player gets a "You win!" message; otherwise, the player gets feedback that encourages them to improve.
  • Target Values: I set a target number of moves (20) and target time (60 seconds) as the baseline for the player to try to beat.

Challenges Faced and Solutions

  • Randomizing the Board: Ensuring a solvable configuration for the board was a challenge. I solved it by starting with an all-white grid and then randomly toggling squares, ensuring a solvable configuration.
  • Timer and Move Counter: Managing the synchronization of these elements was a challenge. However, I ensured that they were properly updated with each player action, allowing for smooth progress tracking.
  • Responsive Design: To ensure that the game works across all screen sizes, I used flexible layouts with CSS grid for the game board. This ensures the game stays usable and visually appealing on any device.
  • Target Values: Determining the right target values required some trial and error. I set target values based on game difficulty, ensuring a balance between fun and challenge.

Insights Gained

  • JavaScript's Power: JavaScript enabled me to build an interactive game with minimal HTML and CSS.
  • Game Design Principles: Balancing difficulty with player progression is crucial in game design. Adding target values creates additional challenges that encourage players to keep playing to improve their scores.
